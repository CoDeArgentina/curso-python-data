def desarrollos_anuales(df, duracion, unidades,idx):
    '''
    Distribuye unidades residenciales a lo largo de
    los aos de construccin del proyecto.
    ...
    Argumentos:
        df(dataframe): dataframe de proyectos residenciales 
        duracion(str): nombre de columna
        unidades(str): nombre de columna
        idx(str): nombre de columna
    Devuelve:
        dataframe: unidades anualmente distribuidas
    '''
    
    df['aos'] = df[duracion].map(lambda x: x/12)
    df['anuales'] = df[unidades].map(lambda x: x/2)
    
    new_df = np.repeat(df[idx], df['aos']).reset_index()
    new_df['inicio'] = 2020
    ao = lambda v: [v+1 if i%2!=0 else v for i,v in enumerate(new_df['inicio'])]
    new_df['inicia'] = ao(new_df['inicio'])
    new_df['finaliza'] = new_df['inicia'] + 1
    
    oferta = np.repeat(df['unidades'].map(lambda x: x/2),df['aos']).\
                       reset_index()['unidades']
    new_df['unidades'] = oferta
    new_df.drop(columns='inicio',inplace=True)
    
    return new_df.iloc[:,1:]

Parte 2: Geopandas, ingresando al mundo de los atributos geo¶

Para ponerlo en trminos extremadamente sencillos, podramos decir que la nica diferencia entre pandas y geopandas es la posibilidad de manipular geometras. Las nociones de Serie y DataFrame sobre las que estuvimos trabajando no desaparecen. Este esquema de manejo de datos (columnas, filas, indexacin, etc.) se mantiene, aunque incorporando un conjunto de mtodos que sirven para operar exclusivamente sobre el principal atributo de un dato espacial: su geometra.

Esta no es otra cosa que una columna ms de nuestra tabla de datos, una Serie dentro de un DataFrame...o mejor dicho, una GeoSerie de un GeoDataFrame. Con ella podremos realizar, de forma muy sencilla, operaciones tales como calcular el rea de un polgono, estimar la distancia de un punto a otro o cambiar de un sistema de proyeccin de coordenadas a otro.

GeoSeries de qué?¶

Como acabamos de decir, 'Geopandas' extiende las funcionalidades de pandas, habiliatando un conjunto de operaciones nuevas sobre objetos geomtricos. Estas operaciones, espaciales o geomtricas, son posible gracias a 'Shapely', el paquete con el que dispone geopandas para realizar manipulaciones geomtricas.

Los objetos de Shapely que componen una GeoSerie son fundamentalmente tres. Puntos, lneas y polgonos. Cada uno de ellos, implementado a partir de una clase propia: Point, LineString/LinearRing y Polygon respectivamente. Tambin es importante aclarar que Shapely cuenta con clases especficas para operar sobre conjuntos o colecciones de cada uno de estos objetos. As es que tambin contamos con las clases de tipo MultiPoint, MultiLineString y MultiPloygon.

Cada uno de estos tres objetos se encuentra asociado con un conjunto de puntos en el plano: en su interior, sus lmites y por fuera de s mismo. Veamos un poco qu significa esto para cada uno de ellos.

1. Puntos

La clase Point posee una dimensin topolgica de cero. Esto quiere decir, que no contiene puntos en sus conjuntos interiores y exteriores y tampoco en sus lmites. En otras palabras, un punto no es ms que un par de coordenadas en el espacio. Una latitud y una longitud.

Tambin por eso es que existen distintas proyecciones. Porque cada una conserva mejor cierto tipo de propiedad antes que otra. Una proyeccin bastante comn es la que se conoce como cilndrica. Esta lleva los meridianos y los paralelos a un cilindro tangente al ecuador del globo. Sin dudas, la ms conocida es la proyeccin Mercator. Esta dispone los meridianos en lneas rectas verticales y los paralelos en lneas rectas horizontales. Ambos perpendiculares entre s, conformando un sistema de coordenadas de tipo cartesiano. Al ser tangente al ecuador deforma las distancias entre los meridianos, aumentando su ancho real cada vez ms a medida que se acerca a los polos (razn por la que no suele ser muy empleada en mapas de latitudes extremas).

Adems de Mercator otra proyeccin cilndrica es la Tranverse Mercator o UTM (de Universal). Este sistema, se caracteriza por apoyar el cilindro de proyeccin sobre el meridiano (en lugar del ecuador). Una variacin de este sistema es el que se utiliza en nuestro territorio nacional: la proyeccin Gauss Krger. Esta es una de las reconocidas y utilizadas por el Instituto Geogrfico Nacional - IGN. Pods consultar algo ms de informacin en su web.

Este sistema de proyeccin divide a la Argentina en 7 fajas meridianas numeradas de oeste a este. Cada faja de la grilla Gauss-Krger mide 3 de ancho (longitud) por 34o de largo (latitud) y tiene como propio origen la interseccin del Polo Sur con el meridiano central de cada faja. Con el objeto de evitar coordenadas negativas, se le asigna al meridiano central de cada faja el valor arbitrario de 500.000 metros y al Polo Sur el valor cero metros.

def point_to_line(gdf,idx):
    '''
    Convierte una Point geometry en LineString
    a partir de un set de coordenadas de inicio y fin.
     ...
    Argumentos:
        gdf(gdf): GeoDataFrame de puntos 
        idx(int): Posicin de cada geometra dentro del gdf. 
    Devuelve:
        LineString
    '''    
    x_inicio = gdf.iloc[idx].geometry.x
    y_inicio = gdf.iloc[idx].geometry.y
    
    try:
        x_fin = gdf.iloc[idx+1].geometry.x
        y_fin = gdf.iloc[idx+1].geometry.y
    except:
        x_fin = gdf.iloc[idx-idx].geometry.x
        y_fin = gdf.iloc[idx-idx].geometry.y
    
    I = (x_inicio, y_inicio)
    F = (x_fin, y_fin)
    
    new_geometry = LineString([I,F])
    return new_geometry
